# Week 5 – Activity 3: Clinic System - Appointment Booking and Medication Order


[![GitHub Repository](https://img.shields.io/badge/GitHub-Repository-blue)](https://github.com/eirikrbe/MSE800-PSD/tree/main/W5/W5Act3)


This is an Activity Diagram design for a basic Clinic System, where a patient can book an appointment, pay for it, and order medication after the prescription is validated. The patient also needs to pay for the medication.


![W5Act3](W5Act3.drawio.png)

---

## Main Features

The Clinic System includes the following features:

- Patient registration and login
- Appointment request with doctor and date selection
- Appointment availability checking
- Booking fee calculation
- Booking payment processing
- Appointment confirmation
- Medication order request
- Prescription validation
- Medication cost calculation
- Medication payment processing
- Medication dispatch through the pharmacy

---

## System Actors

The system has three main actors:

### Patient

The patient can log in or register, request a consultation, pay the booking fee, request medication, pay the medication cost, and receive the medication order.

### System

The system checks appointment availability, reserves appointment slots, calculates fees, processes payments, validates prescriptions, confirms appointments, and sends medication requests to the pharmacy.

### Pharmacy

The pharmacy receives valid medication requests and dispatches the medication order to the patient.
