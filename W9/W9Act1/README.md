# Ako Kupu – Waterfall Project Management Diagram

## Overview

This diagram presents the five-week Waterfall project management plan for **Ako Kupu**, a web-based Te Reo Māori flashcard and learner progress system developed for **MSE800 Assessment 2** 

**DEVELOPMENT TEAM**

Group E

- Yirong Chen
- Eric Gomez


The diagram shows the project moving through six clear phases: requirements analysis, system design, implementation, testing, deployment, and maintenance. Each phase includes its planned timeframe, key activities, and expected deliverables.

## Project Timeline

| Phase                 | Timeframe             | Main Focus                                                                                                                |
| --------------------- | --------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| Requirements Analysis | Week 1, Days 1–5      | Define user stories, acceptance criteria, cultural requirements, and backlog priorities                                   |
| System Design         | Week 2, Days 6–10     | Design the OOP structure, database schema, Flask routes, and user interface wireframes                                    |
| Implementation        | Weeks 3–4, Days 11–24 | Build the Flask web app, domain classes, database integration, authentication, frontend, and Docker setup                 |
| Testing               | Week 4, Days 21–28    | Test OOP classes, user workflows, authentication, route protection, and acceptance criteria                               |
| Deployment            | Week 5, Days 29–35    | Deploy the application to Render, configure environment variables, verify Supabase connection, and finalise documentation |
| Maintenance           | Post-delivery         | Monitor the application, fix defects, gather feedback, and document future improvements                                   |

## Purpose of the Diagram

The purpose of this diagram is to provide a clear project management view of how the Ako Kupu system will be planned, developed, tested, deployed, and reviewed. It helps the development team track progress, manage responsibilities, and connect each project phase to specific deliverables.
