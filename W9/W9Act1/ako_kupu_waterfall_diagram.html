The Waterfall Model is a sequential software development methodology where each phase must be fully completed before the next begins, producing defined deliverables that feed forward — providing clear milestones and traceable progress throughout the three-week development cycle.

1
Requirements analysis
Week 1 · Days 1–3
Define user stories for Ākonga, Kaiako, and Admin
Identify system and cultural requirements
Establish acceptance criteria per user story
Prioritise product backlog on GitHub Projects
Deliverable — User stories, acceptance criteria, prioritised backlog
↓
2
System design
Week 1 · Days 4–7
Design OOP class hierarchy (User, Flashcard, Session)
Design PostgreSQL schema for Supabase
Plan Flask route structure and application layers
Produce class diagrams and UI wireframes
Deliverable — Class diagrams, database schema, architecture plan
↓
3
Implementation
Weeks 1–2 · Days 7–16
Build OOP domain layer and Flashcard subclasses
Develop Flask routes, templates, Bootstrap frontend
Integrate Supabase PostgreSQL and Flask-Login auth
Containerise the application with Docker
Deliverable — Working Flask application running in Docker locally
↓
4
Testing
Weeks 2–3 · Days 14–19
Write unit tests for OOP classes and services
Perform acceptance testing per user story
Test role-based route protection and auth flows
Document test results and resolve defects
Deliverable — Test report, resolved defects, tested prototype
↓
5
Deployment
Week 3 · Days 18–21
Deploy containerised app to Render from GitHub
Configure environment variables and Supabase connection
Verify live URL and all user workflows in production
Finalise README and project documentation
Deliverable — Live deployment URL, GitHub repository, full documentation
↓
6
Maintenance
Ongoing · Post-delivery
Monitor application performance on Render
Address bug reports and vocabulary corrections
Gather learner feedback for future improvements
Document lessons learned in the reflection report
Deliverable — Reflection report, maintenance log, improvement proposals
